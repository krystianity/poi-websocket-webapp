POI Project 1FROEC53 Mainpage

Please note: Spamming requests on the server (120req+/30sec) will result in a short temporary block; to keep the server safe.

Features:

- Object Orienten Approach (incl. wrappers for socket.io and leaflet.js) -

- On Map Application (full controll inside of popups on map) [toHtml() and fromHtml()] -

- Content of markers is loaded on demand (popup open), range of markers is loaded on map movement/panning/zooming -

- First Party App synchronises markers in real time with other visitors (Full-Duplex Websocket Connection and POI Events) -

- Websocket server with portable SQLite3 POI database -

- Proxy server that transfers HTTP POST Requests through a socket connection to the Websocket server -

- Third Party App that exploits the OOM and uses its own php.script to CURL POST Requests to the Proxy server -

- POI Filter + realtime auto complete filter input -

- Open Popups will stop the panning to update the marker list (until they are closed) -

- Simple click somewhere on the map to add a new POI -

- Will automatically get the devices geo position and update the map area on Mobile and Desktop devices -

- Works on mobile devices, make sure to pinch in on a non map area to have easy access to the popups -

- Implemented but not executed in apps: user login, circle, polygon wrapper a.m. -
